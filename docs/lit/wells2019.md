# Wells (2019) — Breaking the Cybernetic Code: The Human Metacognitive Control System

*Frontiers in Psychology 10:2621. Han & Gu 챕터의 ref [12].*
*검토: 2026-07-07. 정독: pp.1–6 (S-REF 개요, CAS, MCS 구조). pp.7–16(치료 함의)은 미정독.*

## WHY / HOW / WHAT

- **WHY**: 심리 장애의 원인을 bottom-up 편향이 아닌 **top-down 자기조절 제어의 부적응**으로 설명하려는 S-REF 전통(1994–)의 이론적 심화.
- **HOW**: Metacognitive Control System(MCS) 제안 — **comparator**가 인지 시스템의 현 상태를 내부 reference model과 비교, 불일치(오차) 검출 시 제어 명령 발행. 상태를 "cybernetic code"로 부호화, monitoring(CS→MCS)·control(MCS→CS)의 양방향 흐름, cybernetic looping(목표 미충족 시 처리 반복).
- **WHAT**: 장애의 보편 기제 = **CAS**(Cognitive Attentional Syndrome): worry·rumination·**threat monitoring**의 지속적(perseverative) 부정 처리. CAS는 metacognitive beliefs — 선언적("worry는 유용하다"/"통제 불능이다")·절차적(worry 생성 "프로그램") — 에 의해 유지됨. 메타분석 45편(Sun 2017)에서 역기능적 메타인지 신념이 장애 전반에 걸쳐 상승; CAS–정신증 연관도 체계적 리뷰로 확인(Sellers 2017).

## 백로그 반영

- **B-05 강화**: 외부 채널 위협 확증 모드의 심리학적 이름 = threat monitoring (CAS). 구현 시 이 용어로 anchoring.
- **[신규 B-09 후보, P3]** `r`을 metacognitive belief 의존 파라미터로 해석하는 주석 (치료 = 파라미터 변경이라는 해석층). Phase 4에서 "시스템 프롬프트/파인튜닝 = belief 변경"과 연결 가능.
